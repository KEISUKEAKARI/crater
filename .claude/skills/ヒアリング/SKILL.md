# Web制作 - ヒアリング＆要件まとめ

クライアントからざっくりした情報をヒアリングして、サイトマップ設計へつなげます。

## 流れ

1. **ヒアリング** - 5つの質問で最小限の情報を聞く
2. **要件まとめ** - 聞いた情報を整理
3. **サイトマップ提案** - 次ステップの提案

---

## ヒアリング（5つの質問）

以下の5項目について、クライアントから情報を聞き出してください。

### 質問テンプレート

```
1. 【サイト種別】
   何のサイトを作りたいですか？
   （例：コーポレート / LP / EC / ポートフォリオ / ブログ など）

2. 【主な目的】
   このサイトで何を実現したいですか？
   （例：集客 / ブランディング / 販売 / 問い合わせ獲得 など）

3. 【ターゲット】
   誰に届けたいサイトですか？
   （例：30代のビジネスマン / 学生 / 企業の採用担当者 など）

4. 【ゴール（CV）】
   ユーザーに最終的にしてほしいアクションは？
   （例：問い合わせ送信 / 資料請求 / 購入 など）

5. 【現在の素材状況】
   以下のうち、既に用意されているものは？
   - テキスト（会社説明・サービス説明など）
   - 写真・画像
   - ロゴ
   - その他
```

---

## 要件まとめ

聞いた情報をもとに、以下の形式で要件をまとめてください。

### 要件まとめシート

```
【プロジェクト概要】
- サイト種別：（ユーザーの回答）
- 制作目的：（ユーザーの回答）
- ターゲット：（ユーザーの回答）
- ゴール（CV）：（ユーザーの回答）
- 素材状況：（ユーザーの回答）

【次のステップ】
次は「サイトマップ設計」で、サイト全体の構造を決めていきます。
以下の項目があれば、さらに詳しく教えてください：
- 参考になるサイト（あればURL）
- 具体的に掲載したいページ（あれば）
- その他の要望・指示

確認：この要件でよろしいですか？
```

---

## サイトマップ設計へのつなぎ

要件確認後、次のステップを提案してください。

### 次ステップの提案

```
ご確認ありがとうございます。

次は【Phase 2: サイトマップ設計】に進みます。

サイトマップとは、サイト全体の構造（どんなページが必要で、どう繋がるか）を視覚化したもので、これが決まると制作がスムーズになります。

下記の情報があれば、より正確なサイトマップが作れます：
- 参考になるサイトURL（あれば）
- 「このページは絶対必要」というページ（あれば）

では、サイトマップを提案させていただきます！
```

---

## ルール

- 聞く情報は **最小限** に。追加情報は後で詳しく聞く
- クライアント確認後、すぐ次フェーズへ進める設計
- 各フェーズ間に「クライアント確認」を挟めるようにする
