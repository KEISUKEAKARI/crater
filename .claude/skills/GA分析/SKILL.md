# Google Analytics 解析スキル

Google Analytics 4（GA4）のデータを取得・分析して、**現状サマリー・課題発見・改善提案**をまとめたレポートを作成します。

---

## 使用ツール（GA4 MCP）

GA4 MCPサーバーが提供するツールを使用します。

| ツール名 | 取得できる情報 |
|---------|-------------|
| `runReport` | カスタムクエリ（自由な指標×ディメンション） |
| `getActiveUsers` | ユーザー数・アクティブユーザー数 |
| `getPageViews` | ページビュー数・人気ページ |
| `getUserBehavior` | セッション時間・直帰率 |
| `getEvents` | イベント・コンバージョン数 |

---

## 手順

### Step 1: 対象期間の確認

ユーザーに分析対象の期間を確認します（指定がなければ過去30日をデフォルトに）。

```
分析する期間を教えてください。
（例：過去30日、先月、今月、過去3ヶ月 など）
指定がなければ「過去30日」で進めます。
```

### Step 2: データ取得（並行して実行）

以下のデータを並行取得する。エラーが出たツールはスキップして取得できた分で進める。

1. **ユーザー・セッション概要**
   ```
   getActiveUsers で直近30日のユーザー数・セッション数を取得
   ```

2. **人気ページ TOP10**
   ```
   getPageViews で上位10ページのPV数・ユニークPVを取得
   ```

3. **ユーザー行動指標**
   ```
   getUserBehavior で平均セッション時間・直帰率・離脱率を取得
   ```

4. **流入元の内訳**
   ```
   runReport で
   dimensions: ["sessionDefaultChannelGroup"]
   metrics: ["sessions", "activeUsers"]
   を取得
   ```

5. **コンバージョン・主要イベント**
   ```
   getEvents でコンバージョンイベントと上位イベントを取得
   ```

### Step 3: レポート作成

取得データをもとに、以下のフォーマットでレポートを作成する。

---

```

データから読み取れる問題点を箇条書きで。
```

---

## データが取れないときの対処

- **MCPが未設定**の場合 → 下記「MCP設定方法」を案内する
- **プロパティIDが不明**の場合 → GA4管理画面での確認方法を案内する
- **権限エラー**の場合 → Google Analyticsへのアクセス権確認を促す
- **一部ツールが使えない**場合 → 使えるツールのみで分析を続ける

---

## MCP設定方法（未設定の場合に案内）

GA4 MCPが設定されていない場合、ユーザーに以下を案内する：

### 簡単な設定方法（Stape.io経由・Googleログインのみ）

1. `~/.claude/settings.json` を開く
2. 以下を追加して保存：

```json
{
  "mcpServers": {
    "ga4": {
      "command": "npx",
      "args": ["-y", "mcp-remote@0.1.30", "https://mcp-ga.stape.ai/mcp"]
    }
  }
}
```

3. Claude Code を再起動
4. Googleアカウントでログイン（ブラウザが自動で開く）
5. 完了！

---

## ルール

- データは必ず**具体的な数値**で示す（「多い」ではなく「○○件」）
- 改善提案は**実行できる具体的なアクション**にする
- 「なぜその改善が効果的か」の**理由**を必ず添える
- 専門用語は使いすぎず、デザイナーにも分かる言葉で書く
- 前期比データがある場合は必ず比較する
- レポートは **日本語** で作成する
