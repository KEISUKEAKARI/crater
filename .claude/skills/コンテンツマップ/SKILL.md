# スキル定義：web-ia（コンテンツマップ作成）

## スキル名
**web-ia** - 情報設計（Information Architecture）/ コンテンツマップ作成

## 目的
LP や Webサイトの1ページ内のセクション構成と要素を整理し、ユーザーの行動フローに沿った情報設計を行うスキル。

## 適用対象
- LP（ランディングページ）制作
- 単一ページサイト
- セクション型ページ設計

## プロセス

### Phase 1: 要件整理
**入力：** サービス企画書/要件ドキュメント
**出力：** 以下の5項目

1. **LPの目的**
   - 何を目指すのか？（問い合わせ獲得、販売、認知など）

2. **ターゲットユーザー**
   - 誰向けか？（年代、職種、企業規模など）

3. **主要セクション案**
   - どんなセクションが必要か？（大体の個数）

4. **参考資料**
   - デザイン参考あるか？（後で分析するか、今すぐするか）

5. **デザイン雰囲気**
   - どんな印象にするか？（モダン、高級感、カジュアルなど）

### Phase 2: コンテンツマップ作成（HTML）
**形式：** HTML ドキュメント

**構成：**
- セクション番号（1-9など）
- セクションタイトル
- セクションの役割
- 各セクションの要素リスト
  - 見出し
  - テキスト説明
  - ビジュアル
  - CTA など

**ユーザージャーニー：** 訪問 → 課題認識 → 解決策 → 信頼構築 → 行動

**格納場所：**
```
/project-root/
  └─ content-map.html（またはsitemap.html）
```

### Phase 3: やり取りで調整
**確認項目：**
- セクションの順序 → OK？
- セクション数 → 多すぎる？少なすぎる？
- 各セクションの要素 → 足りない？不要？
- セクションの役割 → ユーザーを適切に導いている？

**修正方法：**
- HTML で直接修正 → OK ならそのまま使用
- 大きな変更が必要 → Figmaに移行

### Phase 4: 確定
セクション構成が確定したら、次フェーズ（ワイヤーフレーム）へ進む。

---

## 出力ガイドライン

### HTML 構成のポイント
1. **セクション番号**で順序を明確に
2. **セクションの役割**を明記（ファーストビュー、問題認識、解決策など）
3. **ユーザージャーニー**を図示（訪問 → 課題認識 → 解決 → 行動）
4. **各要素を箇条書き**で明確に
5. **プロセス図**があればなお良い

### セクション命名ルール
- 数字 + タイトル（例：「1. ヒーロー」「2. 課題紹介」）
- 役割を括弧で示す（例：「（ファーストビュー）」「（不安解消）」）

---

## テンプレート例

```html
<!-- セクション -->
<div class="section">
  <div class="section-header">
    <div class="section-number">1</div>
    <div class="section-title">セクション名</div>
    <div class="section-type">役割</div>
  </div>
  <div class="section-content">
    <p class="description">このセクションの目的</p>
    <div class="elements">
      <h4>主要要素</h4>
      <ul class="element-list">
        <li><strong>要素1：</strong> 説明</li>
        <li><strong>要素2：</strong> 説明</li>
      </ul>
    </div>
  </div>
</div>
```

---

## 次フェーズ（ワイヤーフレーム）への移行
コンテンツマップが確定したら、各セクションの **視覚的なレイアウト** をワイヤーフレームで設計する。

- セクションの配置
- 要素のサイズ・位置
- ビジュアルとテキストの関係性

---

## 注意点
- **LPは1ページ** → サイトマップ（複数ページの階層図）ではなく、コンテンツマップ（1ページ内のセクション構成）を作成
- **セクション順序が重要** → ユーザージャーニーを妨げない順序にする
- **早めの確認** → コンテンツが決まってからワイヤーに進む方が、修正コストが低い
